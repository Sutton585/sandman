### DesignOps and Design Systems @ Careerstart & WAE

#### Overview
This case study details the creation and implementation of a scalable DesignOps ecosystem within a fast-paced startup environment. As the head of product design and UX, David Sutton's role encompassed both hands-on design and the establishment of a high-functioning DesignOps framework to support rapid iteration, stakeholder alignment, and cross-functional collaboration. The system was built to serve as the backbone of the workflow, from ideation through to execution, maintaining high standards for user experience and product quality.

#### Objective & Challenges
The primary objective was to create a structured and adaptable design environment that supported fast-paced iteration and kept all stakeholders informed and engaged. Key challenges included:
- Disjointed Workflows: Without a structured process, design, engineering, and product teams operated in silos, leading to friction and inefficiency.
- Lack of Transparency: Stakeholders and new team members lacked a centralized source of truth, making it difficult to track design evolution, provide feedback, and understand rationale.
- Inconsistent Design: Without a standardized component library, the app’s user interface lacked consistency, creating a fragmented user experience.

#### Solution
To address these challenges, a comprehensive DesignOps framework was built, organized around several foundational pillars:
- Structured File Organization: A standardized file naming convention and a clear hierarchical page structure in Figma were established. Status indicators (e.g., emojis) were used to provide instant clarity on a file’s purpose and stage.
- Component Library and Atomic Design: A Figma component library was created using atomic design principles, with a precise naming convention for reusability and scalability. This ensured consistency across the app's UI.
- Centralized Documentation Hub: A documentation wiki was created to act as a single source of truth, live-linking to Figma files and housing meeting notes, performance metrics, and UX patterns. This reduced bottlenecks and enabled faster decision-making.
- Integrated Workflows: Figma and Jira were integrated to track project tasks and link design files to corresponding Jira tickets. This created a seamless transition between design and engineering, providing real-time transparency and traceability.

#### Impact
Implementing this DesignOps framework yielded significant, measurable benefits:
- Increased Efficiency: The centralized system reduced the time spent on file organization and communication, allowing the team to focus on quality and rapid iteration.
- Enhanced Consistency: A structured component library and streamlined workflows ensured uniformity across app features, simplifying collaboration between designers and developers.
- Improved Collaboration: The integrated Figma and Jira workflow provided real-time access and transparency, minimizing rework and facilitating agile, user-centered development.
- Seamless Handoffs: A meticulous documentation and handoff process, including granular versioning and centralized handoff files, ensured developers always accessed the most current and approved designs.

#### Lessons Learned
This project demonstrated that a robust DesignOps framework is crucial for scaling a design team and product. It proved that by implementing structured processes, a centralized source of truth, and integrated workflows, a team can operate with agility and alignment, creating a foundation that supports future growth and innovation.

### Results from Validation Testing UI Prototypes at Careerstart

#### Overview
This case study documents the results of validation testing for UI prototypes designed for a job-posting feature at Careerstart. The objective was to address a significant pain point in existing solutions: the inability to quickly post multiple related jobs without redundant data entry. The project focused on creating an interaction flow that supported both short-term and long-term jobs while ensuring compatibility with the Bullhorn CMS system.

#### Objective & Challenges
The primary objective was to validate if a new design solution improved upon the status quo and a previous iteration of the job-posting feature. The core problem was the difficulty of quickly adding related jobs without repetitive data entry. The design also had to accommodate both short-term and long-term jobs while maintaining data consistency with the Bullhorn CMS. Key validation questions included assessing user satisfaction, identifying pain points, and confirming that all user requirements were met by the new design.

#### Solution
The solution involved implementing a new job-posting flow designed for efficiency and ease of use:
- Streamlined Job Posting: A new flow was created where users selected a pre-filled job template rather than typing in all the details from scratch. This eliminated redundant data entry for job titles, descriptions, and other properties.
- Flexible Scheduling: The flow prompted users to specify the job's term length (short-term or long-term), which dynamically determined the subsequent scheduling information they needed to provide.
- Prototyping and Testing: Interactive UI prototypes were used to test the new design with real users. The validation strategy included both qualitative and quantitative methods to compare the new design against the current process.

#### Impact
The validation testing yielded significant, measurable improvements:
- Increased User Satisfaction: Users rated the new WAE process a **9.6 out of 10** for both short-term and long-term jobs, compared to the status quo's ratings of 5.2 and 6, respectively.
- Improved Efficiency: Participants consistently used words like "easy" and "quicker" to describe the new process, with one user noting, "Compared to Wae, working in Bullhorn is like doing your job twice."
- Eliminated Redundancy: The design successfully eliminated repetitive data entry, as users unanimously preferred the predefined shift times over manual entry.
- Actionable Insights: The testing identified new requirements, such as the need for a "qualifications" field for individual job items and the ability to edit requirements on the fly. It also revealed potential opportunities to enable batch-input of jobs via spreadsheets.

#### Lessons Learned
The project proved that a user-centric design focused on eliminating friction and redundancy can lead to dramatic improvements in user satisfaction and efficiency. The validation process was crucial for gathering data-driven insights, confirming the effectiveness of the solution, and identifying key areas for future iteration. It highlighted the value of designing solutions that replicate users' existing behaviors while automating tedious tasks.
